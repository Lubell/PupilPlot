Define Triggers

This menu option allows users to:
  • Define which triggers to segment data around
  • Open and use already existing trigger definitions
  • See which annotations are present within the data
Upon selecting the "Define Triggers" Preprocess Menu option the user is met with two prompts:
  • Shortest or Longest Data file
  • Use already saved trigger definitions - only if trigger definitions already exist
The main window the "Define Triggers" option is comprised of three areas: - A table (on the left side of the window) that lists the triggers/annotations found within the selected subject's data record
  • The left column contains the triggers/annotations as they were saved within the raw data file
  • The right column contains drop downs for each trigger that allows the user to tell the program to either associate the trigger on that line with a certain condition or to ignore it
- Eight windows where each window represents a different experimental condition.
  • Each window is titled (at the start) as Condition A, Condition B, .... , Condition G.
  • After designating which triggers/annotations are associated with a particular condition in the table on the far left of the window, pressing the "Show Conditions by Group" will display all annotations labeled as Condition A in the "Condition A:" text box and will ask the user what the name of the condition is. This will continue on for each condition that has been defined.
    • The name that the user chooses to represent a group of annotations (or condition), will be written into the data record and will show up in the left hand window's first column that displays the trigger names.
    • At any time the conditions can be showed by group and changes or new selections can be made, by pressing the "Show Conditions by Group" button will update the condition boxes and the condition names can be changed or kept.
  • After the user is finished defining the triggers, the "Save Trigger Definitions and Close" should be pressed. This will implement the trigger definitions and PupilPlot, when preprocessing the data, will use these definitions to break the data into condition groups.
    • The trigger definitions should be considered as what denotes a trial
