Shortest or Longest Data file

Because the triggers should be the same across all files, one file's trigger record needs to be selected and used as the representation for all of the other subjects' trigger records.
    • In an ideal universe all of the data records should be the same, but seeing as this isn't typically the case the user must select whether or not the want to use the shortest data record or the longest to define triggers/annotation definitions for the group.
    • It also serves as a good way to find data records that might need to be excluded.
    • If Longest Data record is chosen, PupilPlot will recommend that you delete the shortest record.
    • The selection can always be changed at a later date.
