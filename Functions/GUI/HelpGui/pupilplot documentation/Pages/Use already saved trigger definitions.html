Use already saved trigger definitions

If Define Triggers has already been run and a trigger definition file has already been made and exists in the working directory of the imported eye files, then the user will be queried as to how they would like to proceed.
    • Use trigger definitions as is - Because trigger definitions have already been made this option exists if the user would just like to import the preexisting trigger definitions and bypassing the new trigger definition window.
    • Use trigger definitions and edit - pressing this button will open the trigger definition window with the already defined trigger definitions imported. The user can then make changes or completely alter the definitions.
    • No - ignore the preexisting definitions and essentially begin from scratch.
